# Incident: 2026-04-07 15-43-00

## Summary

```md
At 15:43 UTC on 2026-04-07, an error with the factory occurred where users were no longer able to order pizzas. All requests to the factory were unable to go through. They would recieve the message "Failed to fulfill order at factory" upon attempting to order.

This incident was triggered by chaos monkey. It was detected in the logs for requests to the factory returning a 500 error status code. The sole developer (Gage Smith) was teaching a lab section as a TA when the incident occurred, and started working on it as soon as he was able, which was around 16:50. The issue was resolved at 16:53 by visiting the support link in the error response message.

This was a high severity issue, as 100% of users were unable to order pizzas during the time frame of the incident. During the hour and ten mintues the error persisted, 91 calls to the factory were blocked by the chaos monkey.
```

## Detection

```md
This incident was detected in the logs for factory requests, it displayed a 500 status code with an error message.

It was detected by Gage Smith when reviewing logs at approximately 16:50 UTC.

No automatic alerts were triggered, as a bug in logging code reported all factory logs at 'info' level, regardless of content in the log. This will be fixed in the future for better detecting of errors.
```

## Impact

```md
For 1 hr 10 minutes between 15:43 UTC and 16:53 UTC on 04/07/26, users were completely unable to order pizzas.

This incident affected 100% of JWT-Pizza users. 91 orders were blocked during this time, with an estimated loss of 0.455 ₿.
```

## Root cause

```md
A chaos monkey was set loose, blocking all calls to the pizza factory.
```

## Resolution

```md

```
